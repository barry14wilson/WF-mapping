1 Profile · Safety NetworkNew section on Profile with two rows: Family Group (manage members) and SOS Contact (pick one as primary). Family Group screen lists members with role labels; add-member button opens contacts/SMS invite.
2 Phone Number rowTappable row that opens a phone-entry screen (country code + number + verify CTA). Currently the row doesn't open anything.
3 Back buttonEvery secondary screen has a chevron-back in the top-left. Returns to previous view. Hardware back also goes back, never closes the app.
4 Missing Person flowAdd a fourth pin type "Missing Person" (red). Opens dedicated form with required photo, name, age, description, location. On submit, alert is pushed to every WF user in radius and appears in their Alerts feed with image.
5 Alerts tab — fix crashCurrently the Alerts tab errors. Should render the in-area feed (missing persons, nearby SOS, crime, events) with thumbnail, snippet, distance, age.
6 Reply to alertAlert detail screen shows a message thread with the reporter so finders can share info ("I just saw them at…"). Standard chat input at bottom.
7 SOS targetingSOS goes to Family Group and users in a 2-mile radius — same broadcast pattern as a missing person. Modal shows both audiences before sending.
8 Load / change SOS userDedicated screen lists family members with a radio-style pick. Selected member becomes primary SOS contact (priority call/SMS).
9 Area search resultsTyping an area (e.g. "Camden") in the map search returns an area sheet with the existing Wiley Fox 1–5 safety rating, monthly crime count, category breakdown, and a "Review on map" CTA. Reuses the prototype rating logic.
